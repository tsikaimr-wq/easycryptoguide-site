## 5. 如何运行（本地）

### 5.1 环境要求
- Windows（项目已自带 `server.bat`）
- 可联网（Firebase / TwelveData / EmailJS 需外网）

### 5.2 启动步骤（推荐）
1. 打开终端进入目录：
   `G:\works\AdminecmainPRO\ECPROJECT-main`
2. 运行：
   `server.bat`
3. 浏览器访问：
   `http://localhost:8000`

### 5.3 页面入口
- 官网首页：`http://localhost:8000/index.html`
- 登录页：`http://localhost:8000/login.html`
- 注册页：`http://localhost:8000/signup.html`
- 移动端主应用：`http://localhost:8000/mobile.html`
- 行情页：`http://localhost:8000/markets.html`
- 交割合约页：`http://localhost:8000/delivery_chart.html`
- 后台：`http://localhost:8000/admin.html`

## 6. 部署到国外服务器注意事项
- 这是纯前端静态项目，可直接用 Nginx/Apache/对象存储静态托管。
- 必须在 Firebase Console 把你的海外域名加入 Authorized Domains。
- 若使用手机登录（SMS），reCAPTCHA、Firebase Auth 域名策略需同时放行。
- API Key 当前在前端明文，正式商用建议做网关或权限隔离。

## 7. 建议你上线前做的人工回归
1. 新账号注册（邮箱与手机号各测一次），确认重复注册被拦截。
2. 未 KYC 用户登录后，确认只能进入 KYC 流程。
3. 永续开仓（含 TP/SL）后平仓，核对弹窗和历史数据字段。
4. 后台设置某用户永续结束价后，再平仓验证 P/L 是否按设定结果变化。
5. 行情页抽查多个币种图标是否正常加载。
